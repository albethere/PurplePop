# ADR-011: The Modular Vessel Chassis & Image Factory

## Status
Accepted

## Context
A modern agentic fleet requires a variety of specialized environments (SecOps, Detection, AI Hubs). Relying on a single OS or manual installations is inefficient and creates "Configuration Drift." We need a way to build reproducible, hardened, and "Pulse-Ready" templates for any Linux distribution at the drop of a hat.

## Decision
1.  **The Multi-OS Factory**: `PurplePop` is designated as the primary "Shipyard" for the Fleet. It shall maintain build configurations for multiple OS classes (Ubuntu, Debian, Arch, Kali, etc.).
2.  **Packer-First Pipeline**: We will use HashiCorp Packer as the primary engine to build Proxmox VM templates and LXC images.
3.  **Modular Layering**: Configuration shall be applied via Ansible roles (Modules) that are OS-agnostic where possible.
4.  **Immutable-by-Design**: Templates generated by the factory should be "Immutable," with state stored externally via Mount Points (as per ADR-010).

## Consequences
*   **Interoperability**: Agents can now spin up an "Arch Class" SecOps node as easily as a "Debian Class" server.
*   **Security Baseline**: Every image produced by the factory has a "Gold Master" security posture (CrowdSec/Tetragon) baked in.
*   **Rapid Recreation**: If a Kali-based tactical node is compromised, it is destroyed, and a fresh, updated version is redeployed from the factory in seconds.
